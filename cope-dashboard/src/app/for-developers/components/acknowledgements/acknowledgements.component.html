Acknowledgements

Philosophy

During the conception of COPE, the decision was made to create a concept of an "Acknowledgement". You can think of this almost like an audit. You are able to browse acknowledgements to see

  • What input COPE received
  • What operation COPE did with the input
  • The result COPE came up with based on operation on the provided input
  • The client that submitted the request
  • The time COPE started to work the request
  • The time COPE completed working on the request

Acknowledgements are automatically stored into the COPE database when a request is processed

UUID

A Universally unique identifier (UUID) is provided to a client during each request. This UUID can be used by the support team of the PDMP or the COPE support team in order to investigate a failure to perform an operation

Types

Parameter values and outputs in an acknowledgement have values of anonymous types, this makes them universally applicable for any operation a client may perform. The acknowledgement provides these values in JSON format so that your application is able to serialize and understand

Acknowledgement Viewer

In case it wasn't easily noticeable, there is an acknowledgement viewer in this very dashboard! Just click "Acknowledgements" in the navigation bar to see acknowledgements that COPE has made
